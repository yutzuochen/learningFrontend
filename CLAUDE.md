## Project Overview

This is a Tailwind CSS learning project that uses Vite as the development server and build tool. The project is designed for learning Tailwind CSS fundamentals through hands-on examples.

## Key Technologies

- **Vite**: Modern build tool and development server
- **Tailwind CSS**: Utility-first CSS framework
- **PostCSS**: CSS processing with Autoprefixer

## Development Commands

### Start Development Server
```bash
npm run dev
```
Starts Vite dev server on port 3000 with auto-reload and automatic browser opening.

### Build for Production
```bash
npm run build
```
Creates optimized production build.

### Preview Production Build
```bash
npm run preview
```
Locally preview the production build.

## Project Architecture

### Build Pipeline
The project uses a Vite + PostCSS + Tailwind CSS pipeline:
1. **[src/input.css](src/input.css)**: Source CSS file containing Tailwind directives (`@tailwind base/components/utilities`)
2. **PostCSS**: Processes CSS with Tailwind and Autoprefixer plugins ([postcss.config.js](postcss.config.js))
3. **Tailwind**: Scans HTML files (configured in [tailwind.config.js](tailwind.config.js)) and generates utility classes
4. **[src/output.css](src/output.css)**: Generated CSS file (do not edit manually)

### Configuration Files
- **[vite.config.js](vite.config.js)**: Vite configuration (port 3000, auto-open browser)
- **[tailwind.config.js](tailwind.config.js)**: Tailwind scans `./*.html` files in root directory for class usage
- **[postcss.config.js](postcss.config.js)**: PostCSS plugins configuration

### File Organization
- Root-level HTML files ([index.html](index.html), [navigation-test.html](navigation-test.html), [navigation-practice.html](navigation-practice.html), [dom-tutorial.html](dom-tutorial.html), [test11.html](test11.html)) contain learning examples
- All HTML files use Tailwind utility classes for styling
- [src/input.css](src/input.css) is the entry point for Tailwind CSS

## Important Notes

### CSS Generation
- The [src/output.css](src/output.css) file is auto-generated by Tailwind and should never be edited manually
- Vite automatically processes [src/input.css](src/input.css) during development and production builds
- Tailwind scans all HTML files in the root directory to determine which utility classes to include

### Adding New HTML Files
When creating new HTML files:
1. Place them in the root directory (not in subdirectories) so Tailwind's content scanner can find them
2. Or update `content` array in [tailwind.config.js](tailwind.config.js) to include the new location

### Styling Approach
This project uses Tailwind's utility-first approach exclusively. Avoid writing custom CSS in [src/input.css](src/input.css) unless creating custom Tailwind layers or extending the framework.

## JavaScript Learning Structure

The project now includes comprehensive JavaScript tutorials built progressively from fundamentals to advanced concepts.

### JavaScript Tutorial Files
Located in the root directory, these interactive HTML files teach JavaScript step-by-step:

1. **[js-basics.html](js-basics.html)** - JavaScript Foundation
   - Variables (let, const, var)
   - Data types (String, Number, Boolean, Undefined, Null, Symbol)
   - Operators (arithmetic, comparison, logical)
   - Functions (declarations, arrow functions, default parameters)
   - Conditionals (if/else, ternary operator)
   - Interactive demos: calculator, age checker

2. **[js-arrays-objects.html](js-arrays-objects.html)** - Data Structures
   - Array basics and manipulation
   - Array methods (map, filter, reduce, find, forEach, push, pop, shift, unshift)
   - Object creation and property access
   - Object methods (Object.keys, Object.values, Object.entries)
   - Array of objects (practical real-world patterns)
   - Destructuring assignment
   - Interactive demos: todo list, product management system

3. **[js-dom-events.html](js-dom-events.html)** - DOM Manipulation & Events
   - DOM structure and concepts
   - Element selection (querySelector, querySelectorAll, getElementById)
   - Content manipulation (textContent, innerHTML, value)
   - Style changes (style properties, classList methods)
   - Creating and removing elements dynamically
   - Event listeners (click, input, hover, mouseenter, mouseleave, keypress)
   - Event object properties and methods
   - Interactive demos: click counter, mouse tracker, keyboard input, color picker

4. **[js-forms.html](js-forms.html)** - Form Handling & Validation
   - Getting form values (input, checkbox, radio, select)
   - Form validation patterns and techniques
   - RegEx for email, password, phone validation
   - Real-time validation with immediate feedback
   - Password strength meter
   - Dynamic form field creation
   - Interactive demos: registration form, dynamic contacts list

5. **[js-async.html](js-async.html)** - Asynchronous Programming
   - Synchronous vs asynchronous execution
   - Promises (pending, fulfilled, rejected states)
   - Promise chaining (.then, .catch, .finally)
   - async/await syntax (modern approach)
   - Fetch API for HTTP requests (GET, POST)
   - Handling multiple async operations (Promise.all, Promise.race)
   - Interactive demos: random user API, weather query, concurrent requests

### Learning Path Recommendation
Follow the JavaScript files in this progressive order:
1. js-basics.html → Learn variables, functions, and control flow
2. js-arrays-objects.html → Master data structures
3. js-dom-events.html → Understand browser DOM and user interactions
4. js-forms.html → Build practical form validation
5. js-async.html → Handle API calls and asynchronous operations
